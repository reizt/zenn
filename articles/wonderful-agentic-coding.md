## 🚀 1. はじめに：開発は「指示」と「レビュー」の時代へ

これからの開発者は、エディタを開く必要すらありません。スマホ一つあれば、外出先からでも Discord を通じてバグ修正や機能追加を指示し、プルリクエスト（PR）まで完結させることができます。

この記事では、自宅の **Mac mini** を最強の自律型開発サーバーに変貌させ、**OpenClaw**、**Claude Code**、**Firecrawl** を組み合わせて「スマホから全自動開発」を実現する方法をご紹介します。

## 🏗️ 2. システム構成：24時間働く「物理」エージェント

クラウド完結ではなく、あえてローカルの Mac mini を使うことで、リポジトリ操作の圧倒的な低遅延と、環境構築の自由度を両立させています。

| コンポーネント        | 役割                       | 接続方法                        |
| --------------------- | -------------------------- | ------------------------------- |
| **Discord**           | **スマホからの操作窓口**   | OpenClaw Discord Channel        |
| **Linear**            | タスク管理・実行トリガー   | Webhook (via Cloudflare Tunnel) |
| **OpenClaw**          | **全体の司令塔（脳）**     | Mac mini 上で常駐               |
| **Claude Code**       | **実装・テスト担当（腕）** | ローカルファイルを直接編集      |
| **Firecrawl**         | **最新情報の調査（目）**   | Webスクレイピング連携           |
| **Cloudflare Tunnel** | **外部との安全な架け橋**   | `cloudflared`                   |

承知いたしました。読者の方が実際の手順をイメージしやすいよう、技術的なステップをより具体化し、丁寧な解説を加えた構成案を作成しました。

## 🛠️ 3. セットアップの要所：Mac mini を「開発基地」に変える

自宅の Mac mini を外部と連携させ、AI が自在に動ける「器」を整えるための 3 つのステップを解説します。

### 3.1 🌐 Cloudflare Tunnel で安全な通り道を作る

自宅サーバーを公開する際、最大の壁となるのが「固定IPの確保」や「ルーターのポート開放」です。これらを一切不要にするのが **Cloudflare Tunnel** です。

1. **cloudflared のインストール**: Mac mini で `brew install cloudflare/cloudflare/cloudflared` を実行します。
2. **トンネルの作成**: `cloudflared tunnel create agent-tunnel` でトンネルを生成し、独自のドメイン（例: `agent.example.com`）とローカルの OpenClaw ポート（デフォルトは 3000 番）を紐付けます。
3. **認証の完了**: これにより、Linear からの Webhook 通知や Discord からの信号が、安全に自宅の Mac mini へ届くようになります。

### 3.2 ⚙️ OpenClaw の導入と設定

OpenClaw は、AI エージェントを動かすための「ランタイム（実行基盤）」です。

1. **初期化**: `npx openclaw init` を実行し、基本設定ファイルである `openclaw.json` を生成します。
2. **チャンネル設定**: Discord ボットのトークンを登録し、どのチャンネルで AI と対話するかを指定します。
3. **セキュリティ**: 実行を許可するコマンドのホワイトリストや、AI がアクセスできるディレクトリの範囲を制限することで、安全な実行環境を構築します。

### 3.3 🔨 Claude Code を「スキル」として組み込む

Anthropic が提供する **Claude Code** は、ターミナル上で高度なコード編集を行うツールです。これを OpenClaw の「スキル（道具）」として登録することで、AI が自分の判断でコードを書き換えられるようになります。

```json
{
  "skills": [
    {
      "name": "claude_code",
      "description": "プロジェクトのコードを分析し、バグ修正や機能実装、テストを実行します",
      "command": "claude \"<prompt>\"",
      "working_directory": "/path/to/your/project"
    }
  ]
}

```

この設定により、OpenClaw という「脳」が、Claude Code という「腕」を使ってローカルファイルを直接編集し、`npm test` などのコマンドまで完結させることが可能になります。

### 3.4 🔍 Firecrawl で「最新の知恵」を授ける

AI は学習データのカットオフ以降の最新ライブラリの仕様を知りません。そこで **Firecrawl** を連携させます。
OpenClaw が「このライブラリの最新の書き方がわからない」と判断した際、Firecrawl が自動で公式ドキュメントをクロールし、AI が理解しやすい Markdown 形式に変換して情報を注入します。これにより、「古い知識による誤ったコード」を防ぐことができます。

## 🔄 4. スマホからの開発フロー

この環境が整えば、あなたの開発サイクルは以下のように変わります。

1. **[あなた]** 外出先からスマホで **Linear にチケットを起票**。
2. **[AI]** Webhook を検知して OpenClaw が起動。最新のライブラリ仕様を **Firecrawl** で調査。
3. **[AI]** **Discord** に「修正方針が決まりました。実行しますか？」と通知。
4. **[あなた]** スマホで通知を確認し、**Discord で「承認」を送信**。
5. **[AI]** **Claude Code** がローカルコードを書き換え、テストを通し、GitHub に PR を作成。
6. **[あなた]** GitHub アプリで内容を確認し、マージ。

Mac mini という物理資産を AI エージェントの「体」として活用することで、スマホ一つで開発が進む未来が手に入ります。
